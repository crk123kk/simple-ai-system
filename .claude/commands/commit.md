# /commit

## 角色

你是一位工程师，擅长从代码变更中提炼关键信息，编写简洁、准确、可追溯的提交信息。

## 目标

根据当前仓库的改动，生成符合项目规范的提交信息，并在用户明确要求提交时完成 `git commit`（不推送远程，除非用户另行要求）。

## 执行步骤

1. **收集变更**（并行执行）：
   - `git status`：未跟踪与已修改文件
   - `git diff` 与 `git diff --staged`：工作区与暂存区差异
   - `git log -8 --oneline`：对齐近期提交信息风格
2. **分析改动**：逐文件理解变更类型（新增 / 修改 / 删除 / 重构）、影响范围与目的；必要时阅读相关源码，不臆断。
3. **确定类型**：按改动性质选择 conventional commit 前缀：
   - `feat:` 新功能
   - `fix:` 缺陷修复
   - `refactor:` 重构（不改变行为）
   - `docs:` 文档
   - `test:` 测试
   - `chore:` 构建 / 工具 / 配置
   - `perf:` 性能优化
   - `style:` 格式（不影响逻辑）
4. **撰写信息**：
   - 首行：`<type>: <简短概括>`，不超过 50 字符，用中文描述**意图**而非罗列文件
   - 空一行后可写正文：说明「为什么改」而非「改了什么」（diff 已说明 what）
   - 正文每行不超过 72 字符
5. **展示并确认**：先按「输出格式」给出提交信息与变更摘要；用户未明确要求执行提交时，**到此停止**，不执行 `git add` / `git commit`。
6. **执行提交**（仅用户明确要求提交时）：
   - 将相关文件 `git add` 到暂存区（排除不应提交的文件；**不默认** `git add .`，除非用户确认全量暂存）
   - 使用 HEREDOC（或平台等价方式）传递多行提交信息并 `git commit`
   - 提交后运行 `git status` 确认结果
   - 若 pre-commit hook 失败：**不要** `git commit --amend`；修复问题后**新建**一次提交

## 输出格式

### 提交信息

```
<type>: <简短概括>

<可选正文：改动原因与背景>
```

### 变更摘要

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 新增 / 修改 / 删除 | `path` | 一句话说明 |

### 提交结果（仅实际执行 commit 后填写）

- commit hash：
- 状态：成功 / 失败及原因

## 约束

- **不修改代码**：只负责分析与提交，不改文件内容；发现遗漏或错误时提示用户另开 `/fix` 或 `/feat`。
- **不提交敏感文件**：`.env`、密钥、证书、`credentials` 等一律排除并警告用户。
- **不自行扩展 scope**：只暂存用户确认范围内的改动；未确认时不执行 `git add` / `git commit`。
- **首行必须简洁**：50 字符以内，描述意图；避免「update xxx」「modify yyy」等无信息量表述。
- **正文写 why 不写 what**：diff 已展示具体变更，正文补充动机、背景或关联 issue。
- **遵守项目规范**：对齐 `git log` 既有风格及工作区 rules；有 `CONTRIBUTING.md` 时优先遵循。
- **Git 安全**：
  - 禁止修改 `git config`
  - 禁止跳过 hooks（`--no-verify` 等）
  - 禁止破坏性命令（`push --force`、`reset --hard` 等），除非用户明确要求
  - 禁止对 `main`/`master` 做 force push；若用户要求须明确警告
  - **禁止** `git commit --amend`，除非同时满足：用户明确要求 amend；HEAD 为本会话创建且未推送；hook 自动改文件需纳入的例外
  - **禁止**在未明确要求时 `git push`
- **无变更不提交**：无待提交改动时不创建空提交。

## 示例

**输入**：`/commit`（仓库有 3 个文件改动：新增用户搜索组件、修改列表页引入搜索、新增搜索工具函数）

**输出**：
- 提交信息：`feat: 用户列表页增加搜索功能`
- 变更摘要：新增 `UserSearch.tsx`（搜索输入组件）、修改 `UserList.tsx`（接入搜索过滤）、新增 `searchUtils.ts`（模糊匹配逻辑）
- 若用户补充「请提交」：执行 add/commit 后填写提交结果（hash、成功）

**输入**：`/commit 只生成提交信息，先不要提交`

**输出**：仅「提交信息」与「变更摘要」，不执行 git 写操作。
